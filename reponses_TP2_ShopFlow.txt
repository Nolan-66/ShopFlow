1.2 Pourquoi scope='module' pour la fixture client ?

Le scope='module' permet de créer le TestClient une seule fois par fichier de test, au lieu de le recréer à chaque fonction de test.
Cela accélère l’exécution, car le cycle de vie FastAPI (startup/shutdown) n’est pas relancé à chaque test.
En contrepartie, l’isolation est un peu moins forte qu’avec scope='function', car plusieurs tests d’un même fichier partagent le même client.
Ici, cela reste acceptable car la base de données de test reste isolée et les données sont contrôlées par les fixtures.

1.3 Pourquoi faire assert response.status_code == 201 dans la fixture api_product ?

L’assertion dans la fixture permet de valider immédiatement que la donnée de test a bien été créée.
Si la création échoue, le test s’arrête directement au niveau de la fixture, ce qui évite d’exécuter un test avec un objet invalide.
Cela rend les erreurs plus claires et garantit que tous les tests utilisant api_product partent d’un état cohérent.

4.3 Rôle de Faker('fr_FR')

Faker('fr_FR') génère des données réalistes localisées en français, contrairement à Faker() par défaut qui produit des données plus génériques.
Dans le contexte de ShopFlow, cela est utile pour produire des contenus de test cohérents avec un usage français :
- noms de produits réalistes,
- textes en français,
- données proches du contexte fonctionnel attendu.
Cela améliore la lisibilité des tests et leur réalisme.

5.3 Différence entre test d’intégration et smoke test

Un test d’intégration vérifie que plusieurs couches fonctionnent bien ensemble :
route HTTP, service métier, base de données, sérialisation JSON.

Un smoke test est un test très rapide qui vérifie seulement que l’application est vivante et utilisable :
par exemple /health, /docs ou /openapi.json.

Dans Jenkins, les smoke tests doivent être lancés en premier car ils permettent de détecter très vite un problème de démarrage.
Si les smoke tests échouent, il est inutile d’exécuter toute la suite.
Les tests d’intégration viennent ensuite, car ils sont plus complets et plus coûteux.
